# 하나의 프로그램 안에 B+ 트리가 여러 개 있을 수 있을까?

## 결론부터

`네, 가능합니다.`

하나의 프로그램 안에 B+ 트리가 여러 개 있을 수 있습니다.

## 왜 여러 개가 필요할까?

B+ 트리는 `무엇을 기준으로 빠르게 찾을지` 정해 두는 구조입니다.

그래서 찾는 기준이 여러 개라면
그 기준마다 B+ 트리를 따로 만들 수 있습니다.

예를 들어 플레이어 데이터가 있다면:

- `id` 기준 B+ 트리
- `name` 기준 B+ 트리
- `score` 기준 B+ 트리

이렇게 여러 개를 둘 수 있습니다.

## 쉽게 비유하면

같은 서류철이 하나 있어도
찾는 방법은 여러 가지일 수 있습니다.

- 사번순 색인
- 이름순 색인
- 부서순 색인

즉, 데이터는 하나인데
`찾아가는 길`이 여러 개 있는 것입니다.

## 우리 프로젝트에 연결하면

이번 과제에서는 보통 `id` 기준 B+ 트리 하나를 먼저 만듭니다.

이유는:

- `id`는 중복이 거의 없습니다
- 숫자 비교가 쉽습니다
- `WHERE id = ?` 조건에 바로 연결됩니다

필요하다면 나중에:

- `name` 기준 인덱스
- `score` 기준 인덱스

도 추가할 수 있습니다.

## 여러 개 만들면 좋은 점

- 다양한 검색 조건을 빠르게 처리할 수 있습니다
- 자주 찾는 값을 더 빨리 찾을 수 있습니다

## 여러 개 만들면 생기는 단점

- 데이터를 넣을 때 인덱스도 여러 개 같이 업데이트해야 합니다
- 메모리를 더 사용합니다
- 관리가 더 복잡해집니다

그래서 실무에서는
`자주 사용하는 조건에만 인덱스를 만든다`
가 중요합니다.

## 한 문장 정리

`B+ 트리는 하나만 있는 것이 아니라, 검색 기준이 여러 개면 그 기준마다 여러 개 만들 수 있다.`
