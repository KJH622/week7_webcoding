# 왜 B+ 트리를 쓸까?

## 아주 쉽게 설명하면

B+ 트리는 `엄청 많은 책이 있는 도서관에서 원하는 책을 빨리 찾기 위한 똑똑한 정리 방법`이라고 생각하면 됩니다.

책이 100만 권 있는데 처음부터 한 권씩 다 보면 너무 오래 걸립니다.
이건 `선형 탐색`입니다.

그런데 도서관이 이렇게 잘 정리되어 있다고 생각해 보세요.

- 1번부터 100번 책은 저쪽
- 101번부터 200번 책은 여기
- 201번부터 300번 책은 저기

그러면 우리는 처음부터 끝까지 다 보지 않고,
`큰 묶음 -> 더 작은 묶음 -> 원하는 책`
순서로 빨리 찾아갈 수 있습니다.

이게 바로 트리 구조가 하는 일입니다.

## B+ 트리가 좋은 이유

### 1. 하나를 빨리 찾을 수 있다

예를 들어 `id = 500000`인 플레이어를 찾고 싶다면,
50만 명을 한 명씩 다 볼 필요가 없습니다.

B+ 트리는 중간중간 길잡이 표지판처럼 정리되어 있어서
바로 가까운 곳으로 내려갈 수 있습니다.

## 2. 여러 개를 연속으로 찾기 좋다

예를 들어 `id 250000 ~ 251000`처럼 범위로 찾고 싶을 때,
B+ 트리는 맨 아래 데이터들이 줄줄이 연결되어 있어서
옆으로 쭉 읽으면 됩니다.

그래서 `범위 검색`에 특히 강합니다.

## 그냥 찾는 방법과 비교하면

- 그냥 찾기: 처음부터 끝까지 다 보기
- B+ 트리: 목차와 색인을 보고 바로 근처로 가기

즉, 데이터가 많아질수록 B+ 트리가 훨씬 유리합니다.

## 우리 프로젝트에 연결하면

우리 프로젝트는 게임 플레이어 데이터가 아주 많다고 가정합니다.

- 플레이어가 100명 정도면 그냥 찾아도 괜찮습니다.
- 플레이어가 100만 명이면 처음부터 끝까지 찾는 건 너무 느립니다.

그래서 `id` 같은 번호를 기준으로
미리 B+ 트리에 정리해 두고 빠르게 찾으려는 것입니다.

## 한 문장 정리

`B+ 트리는 데이터가 아주 많을 때, 원하는 값을 처음부터 끝까지 보지 않고 빠르게 찾게 해주는 정리 방식이다.`
